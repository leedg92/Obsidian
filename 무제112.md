# 회원 관련 API 가이드 (Postman 테스트용)

> 기본 경로: `http://{서버주소}/allcone-api`
> Content-Type: `application/json`

---

## 1. SMS 본인인증

### 1-1. 인증번호 발송 요청

```
POST /mobile/auth/req
```

**Headers:** 없음

**Body:**
```json
{
  "userHp": "01012345678"
}
```

**성공 응답:**
```json
{
  "code": "OK",
  "message": null,
  "data": null
}
```

---

### 1-2. 인증번호 확인

```
POST /mobile/auth/res
```

**Headers:** 없음

**Body (회원가입용 - SIGN_UP):**
```json
{
  "userHp": "01012345678",
  "authNo": "123456",
  "userName": "홍길동",
  "authType": "SIGN_UP"
}
```

**Body (아이디찾기 - FIND_ID):**
```json
{
  "userHp": "01012345678",
  "authNo": "123456",
  "authType": "FIND_ID"
}
```

**Body (비밀번호찾기):**
```json
{
  "userId": "testuser01",
  "userHp": "01012345678",
  "authNo": "123456",
  "authType": "FIND_PWD"
}
```

**Body (전화번호변경 - CHANGE_HP):**
```json
{
  "userId": "testuser01",
  "userHp": "01099998888",
  "authNo": "123456",
  "authType": "CHANGE_HP"
}
```

> 비운영 환경(dev/local)에서는 인증번호 `"111111"` 로 우회 가능

**성공 응답 (SIGN_UP, 미등록 번호일 때):**
```json
{
  "code": "OK",
  "message": null,
  "data": {
    "userName": "홍길동",
    "userHp": "01012345678",
    "driverName": "홍길동",
    "driverHp": "01012345678"
  }
}
```

**성공 응답 (FIND_ID 등):**
```json
{
  "code": "OK",
  "message": null,
  "data": {
    "userId": "testuser01",
    "userName": "홍길동",
    "userHp": "01012345678",
    "driverId": "testuser01",
    "driverName": "홍길동",
    "driverHp": "01012345678",
    "joinDate": "2024-01-15",
    "isDriver": true
  }
}
```

**authType 정리:**

| authType | 용도 | 조회 키 |
|----------|------|---------|
| `SIGN_UP` | 회원가입 (중복 번호 체크) | userHp |
| `FIND_ID` / `SEARCH_ID` | 아이디 찾기 | userHp |
| `CHANGE_HP` | 전화번호 변경 | userId |
| 그 외 (ex: `FIND_PWD`) | 비밀번호 찾기 등 | userId |

## 2. 아이디 중복 확인

```
POST /mobile/signup/exists
```

**Headers:**
| Key | Value | 필수 |
|-----|-------|------|
| deviceToken | {FCM 디바이스 토큰} | O |

**Body:**
```json
{
  "userId": "testuser01"
}
```

**성공 응답 (사용 가능):**
```json
{
  "code": "OK",
  "message": null,
  "data": null
}
```

**실패 응답 (중복):**
```json
{
  "code": "ERROR",
  "message": "이미 등록된 아이디입니다.",
  "data": null
}
```

## 3. 회원가입

```
POST /mobile/signup
```

**Headers:**
| Key | Value | 필수 |
|-----|-------|------|
| deviceToken | {FCM 디바이스 토큰} | O |

**Body:**
```json
{
  "userId": "testuser01",
  "password": "password123!",
  "userName": "홍길동",
  "userHp": "01012345678",
  "userRole": "ROLE_DRIVER",
  "agreeMarketing": "Y",
  "recommendTruckNo": "부산12가3456",
  "recommendName": "김추천",
  "recommendUserId": "recommend01",
  "serviceTermsAgreement": "Y",
  "personalInfoProcessingPolicyAgreement": "Y",
  "personalLocationInfoProcessingPolicyAgreement": "Y",
  "locationInfoBusinessTermsAgreement": "Y",
  "userBirthDate": "1990-01-15T00:00:00"
}
```

> `recommendTruckNo`, `recommendName`, `recommendUserId`, `agreeMarketing`은 선택 항목

**성공 응답:**
```json
{
  "code": "OK",
  "message": null,
  "data": null
}
```

## API 호출 순서 (일반적인 회원 플로우)

```
1. SMS 인증번호 발송     POST /mobile/auth/req
2. SMS 인증번호 확인     POST /mobile/auth/res  (authType: SIGN_UP)
3. 아이디 중복 확인      POST /mobile/signup/exists
4. 회원가입             POST /mobile/signup
5. 디바이스 토큰 확인    POST /mobile/checkusable/v2
6. 로그인              POST /mobile/signIn/v2
7. (이용약관 동의)      POST /mobile/user/updateTermsAgree
8. 로그아웃            POST /mobile/signOut/v2
```

## 참고

- 모든 응답은 `ApiResponse` 형태: `{ "code": "OK"|"ERROR", "message": "...", "data": {...} }`
- `userId` / `driverId` 는 하위호환을 위해 둘 다 수용 (userName/driverName, userHp/driverHp 도 동일)
- 비운영 환경 SMS 인증 우회: 인증번호 `"111111"` 사용
- 비밀번호는 서버에서 eGov PasswordEncoder로 암호화 저장
- 전화번호(`userHp`)는 TextCryptoService로 암복호화
- 생년월일(`userBirthDate`)은 암호화 저장, 로그인 시 복호화하여 응답
