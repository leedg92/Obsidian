# 회원 탈퇴 및 인증·마스킹 정책

본 문서는 회원 탈퇴 처리 방식, 비밀번호 재설정 시 반영 항목, 로그인 ID·비밀번호 복잡도, 그리고 개인정보 마스킹 정책을 정리한 참고 자료입니다.

---

## 1. 회원 탈퇴 처리 개요

### 1.1 탈퇴 처리 경로

회원 탈퇴는 다음 **두 가지 경로**에서 수행됩니다.

| 구분 | 설명 |
| --- | --- |
| 포털(본인) | 이용자가 포털에서 직접 탈퇴하는 기능 |
| 관리자 | 관리자 화면에서 해당 회원을 탈퇴 처리하는 기능 |

두 경로 모두 아래 **공통 규칙**에 따라 `tb_member` 등 회원 정보를 동일한 방식으로 갱신합니다.

---

## 2. 탈퇴 시 `tb_member` 공통 갱신 항목

탈퇴 시점에 아래 컬럼(필드)을 일괄 반영합니다.  
`MBR_ID`는 회원 PK이며, 일부 필드에는 **PK와 동일한 값** 또는 **PK를 암호화한 값**을 저장합니다.

| DB 컬럼 | Entity 필드 | 내용 |
| --- | --- | --- |
| `LGN_ID` | `loginId` | 회원 PK(`MBR_ID`)와 **동일한 값**으로 변경 |
| `PSWD` | `password` | `MBR_ID`를 **암호화**하여 저장 |
| `PSWD_EXPRY_DT` | `passwordExpireDt` | **탈퇴 시각** |
| `MBR_NM` | `name` | 고정 문자열 **`탈퇴회원`** |
| `MBR_EML_ADDR` | `email` | `MBR_ID`를 **암호화**하여 저장 |
| `MBR_TELNO` | `phone` | `MBR_ID`를 **암호화**하여 저장 |
| `TMPR_PSWD_YN` | `tempPasswordYn` | `N` |
| `TERM_AGRE_YN` | `termsAgreeYn` | `N` |
| `PRVC_AGRE_YN` | `privacyAgreeYn` | `N` |
| `OVR14_AGRE_YN` | `over14AgreeYn` | `N` |
| `MRKTN_EML_RCPTN_AGRE_YN` | `marketingEmailAgreeYn` | `N` |
| `MRKTN_EML_RCPTN_AGRE_DT` | `marketingEmailAgreeDt` | **탈퇴 시각** |
| `MRKTN_SMS_RCPTN_AGRE_YN` | `marketingSmsAgreeYn` | `N` |
| `MRKTN_SMS_RCPTN_AGRE_DT` | `marketingSmsAgreeDt` | **탈퇴 시각** |
| `LOCTN_TERM_AGRE_YN` | `locationTermsAgreeYn` | `N` |
| `LOCTN_TERM_AGRE_DT` | `locationTermsAgreeDt` | **탈퇴 시각** |
| `LCK_YN` | `lockYn` | `N` |
| `USE_YN` | `useYn` | `N` |
| `WHDWL_DT` | `withdrawalDate` | **탈퇴 시각** |
| `CI_VAL` | `ciVal` | **`null`** |
| `CI_HASH` | `ciHash` | **`null`** |

> **참고:** 암호화 알고리즘·키 관리 등 구현 세부사항은 시스템 보안 정책 및 백엔드 표준을 따릅니다.

---

## 3. 비밀번호 재설정 시 처리 항목

비밀번호를 재설정할 때는 다음 항목을 갱신합니다.

| 항목 | 처리 내용 |
| --- | --- |
| 비밀번호 | 신규 비밀번호로 변경(저장 시 암호화 등 기존 정책 적용) |
| 비밀번호 만료일 | **재설정 시점 기준 6개월 후** |
| 잠금 여부 | `N` |
| 로그인 실패 횟수 | `0` |
| 수정자 | 재설정 처리 주체에 따른 수정자 반영 |
| 수정일시 | 재설정 처리 시각 |

---

## 4. 로그인 ID 복잡도

| 항목 | 규칙 |
| --- | --- |
| 허용 문자 | **영문·숫자만** |
| 길이 | **3자 이상 25자 이하** |
| 비허용 | 한글, 공백, 특수문자 **불가** |

---

## 5. 비밀번호 복잡도

| 항목 | 규칙 |
| --- | --- |
| 길이 | **9자 이상 21자 이하** |
| 허용 문자 | 영문(`a-zA-Z`), 숫자(`0-9`), 아래 특수문자만: **`!` `@` `#` `$` `%` `^`** |
| 조합 | 위 **세 가지 유형(영문, 숫자, 특수문자)을 모두 포함** |

---

## 6. 마스킹 처리 정책

| 구분 | 정책 |
| --- | --- |
| 일반 조회·응답 | **이름**의 **가운데 글자 1글자만** 마스킹 처리 |
| 아이디 찾기 등 | 해당 흐름에서는 **마스킹을 적용하지 않음** |

연동 시스템·화면에서 동일 정책을 맞출 경우, 위 기준을 기본으로 합니다. 세부 예외(관리자 전용 화면, 법적 보관 데이터 등)가 있으면 별도 보안·개인정보 처리 방침과 합의합니다.
