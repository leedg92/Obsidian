## 회의록 요약

**회의 주제:** 포트넥스 / 체인포털 / NICE 인증 / DB 동기화

---

### 1. 체인포털 DB

- DDL 및 데이터는 혼합된 스키마 조회로 확인 가능하므로 별도 수령은 급하지 않음
- 나머지 DDL 및 데이터 관련 문의 필요 시 → **조대환 책임(유지보수팀)에게 요청**

---

### 2. 구 DB ↔ 신 DB 동기화

- `tb_member` insert → 특정 컬럼값 조건 만족 → 트리거 실행 → 동기화
- **포트넥스 측에서 트리거 실행 조건값 별도 전달 필요**

---

### 3. 포트넥스 DB 배치 연동

- 현재 배치 작업으로 올컨e DB 테이블 데이터 수집 중
- 탈퇴 회원 처리 일부 하드코딩 상태 → **포트넥스 측 탈퇴 회원 처리 정책 전달 필요**
- insert 로직 반영 예정이며, 기존 배치 동작과 혼용 시에도 영향 없을 것으로 판단

---

### 4. 운영망 / 개발망 DB 차이

- 운영망 ↔ 개발망 간 테이블 차이 다수 존재
- 계정별로 운영망 조회 가능 테이블이 상이함 → **정합성 확인 필요**

---

### 5. Allcone NICE 인증

| 항목             | 내용                                      |
| -------------- | --------------------------------------- |
| V1 만료          | 인증 완료 시 삭제, expire = 30분                |
| V2 만료          | 일 배치 삭제, expire = 10분                   |
| 만료 처리 (스마트엠투엠) | expire 경과 시 토큰 초기화 → NICE 토큰 불필요 재발급 방지 |
| V1 정리 일정       | 포트넥스 4월 내 V1 정리 예정, V2 전환 중 지속 소통       |
| niceCert 테이블   | 복호화된 인증 결과 저장 중, 향후 배치로 삭제 예정           |
| 인증 방식          | SMS 인증 + NICE 인증 혼합 구조 당분간 유지           |


---

### 6. AAS

- 체인포털 DB AAS 관련 테이블 **컬럼명 변경 불필요** (조대환 책임 판단)
    - 단, **신승민 사원에게 추가 확인 예정**
- `/validate` API: `bizNo` → `BR_NO` 변경 예정
    - **조해인 프로: 두 파라미터 모두 처리 가능하도록 수정 예정**
- AAS 모듈 배포: **블록체인 배포와 함께 진행**
- **오경우 선임 → 사업단:** 모듈별 AAS 호출 부분 확인 요청 예정
