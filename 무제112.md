
## conType 패턴

| 그룹          | 형태       | 예시                          | 비중    | 파싱           |
| ----------- | -------- | --------------------------- | ----- | ------------ |
| ① 4자리 숫자    | `NNNN`   | `4510`,`2210`,`4500`        | ~95%  | 사이즈+타입       |
| ② ISO표준 영숫자 | `NN`+알파벳 | `45R1`,`22G1`,`42G0`,`L510` | ~0.3% | 사이즈+타입       |
| ③ 2글자 알파벳   | `AA`     | `GE`,`RF`,`FR`,`OT`,`TK`    | ~4.6% | 타입만 (사이즈 불가) |
| ④ 노이즈       | -        | `null`(문자열) 등               | 극소수   | 불가           |

## 사이즈 매핑 (con_size) — 앞부분

ISO 6346: 1번째 글자 = 길이. (2번째 = 높이,  높이는 컨테이너 타입과는 무관하므로 데이터는 "길이만" 사용, HC 구분 안 함)

| 1번째 글자                              | con_size           |
| ----------------------------------- | ------------------ |
| `2`                                 | `20` (20ft)        |
| `4`                                 | `40` (40ft, HC 포함) |
| `L`                                 | `45`               |
| `M`                                 | `48`               |
| `N`                                 | `49`               |
| 그 외(`9`,`5`,`G`,`R`,`F`,`O`,`T`...) | `''` (공란)          |


컨테이너 타입은 ISO 6346 형식에 따라서 뒷 2자리만 사용합니다(십의자리)
