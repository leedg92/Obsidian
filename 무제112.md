## API 목록 및 Postman 테스트

> **Base URL**: `http://{테스트서버}:{포트}/allcone-api`
> deviceToken 헤더가 필요한 API는 로그인 후 받은 토큰 사용

---

### A. NICE 본인인증 플로우

#### A-1. NICE 인증 URL 요청
```
GET /allcone-api/mobile/auth/nice/auth-url?purpose={purpose}
```

| purpose 값                    | 용도          |
| ---------------------------- | ----------- |
| `ALLCONE_APP_JOIN`           | 회원가입        |
| `ALLCONE_APP_FIND_ID`        | 아이디 찾기      |
| `ALLCONE_APP_RESET_PASSWORD` | 비밀번호 초기화    |
| `ALLCONE_APP_AUTH_UPDATE`    | 기존 회원 인증 갱신 |

**응답**:
```json
{ "auth_url": "https://auth.niceid.co.kr/..." }
```

#### A-2. NICE 인증 결과 수신 (return_url 콜백)
```
GET /allcone-api/mobile/auth/nice/result?request_no={req_no}&web_transaction_id={txn_id}&purpose={purpose}
```
**응답**:
```json
{ "certId": "UUID" }
```
> 이후 비즈니스 API에서 certId를 사용

## 테스트 순서 (권장)

1. **NICE 인증 URL 요청** (A-1) → auth_url 확인
2. **아이디 중복 확인** (B-1) → 존재하지 않는 ID 확인
3. **회원가입** (B-3) → tb_member, tb_member_authority INSERT 확인
4. **로그인** (C-1) → 정상 로그인 + 응답 확인
5. **세션 확인** (C-2) → usable 확인
6. **비밀번호 확인/변경** (E-1, E-2)
7. **약관 동의 업데이트** (E-4)
8. **차량 목록** (H-1) → tb_member JOIN 확인
9. **로그아웃** (C-3)
10. **아이디 찾기** (D-1) → NICE certId 필요
11. **비밀번호 초기화** (D-2) → NICE certId 필요
12. **회원 탈퇴** (F) → 익명화 확인
