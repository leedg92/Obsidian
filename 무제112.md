# app-allcone ↔ api-bctrans URL 매핑 검증

> 작성일: 2026-04-20
> 브랜치: `feature/bctransApiForAllcone/WIP`
> 목적: 신 allcone 분리 시 bctrans가 열어야 할 provider API 확정을 위한 기초 자료. 앱이 호출하는 allcone-api URL 전수와 api-bctrans Controller 매핑 결과 기록.

## 조사 방법

1. 앱 소스: `/mnt/d/workspace/bctrans_prod/app-allcone/lib/core/blocs/initialization_bloc.dart` RemoteConfig 기본값에서 `/mobile/*`, `/healthCheck/` 등 allcone-api 프리픽스로 호출되는 URL 전수 추출
2. api-bctrans의 `@RequestMapping`, `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping` 전수 검색 후 URL 대조

## 3. 침범 분류 결과

### 분류 조건과 추가 대상 여부

| 분류 | 조건 | 신규 bridge API 추가 대상 |
|------|------|:---:|
| **침범 A** | allcone Service 내부에서 `bctransUrl`로 webClient HTTP 호출 중 (serverName 분기 또는 항상 HTTP) | ❌ 기존 엔드포인트 재사용 |
| **침범 B** | Mapper SQL FROM/JOIN 절에 bctrans 소유 테이블 포함 | ⭕ **신규 필요** (단, `AllconeBridgeController`에 이미 구현된 엔드포인트는 제외) |
| **외부 위임** | Service 로직이 chainportal 외부 API 호출 또는 `chainportaldb.*` 테이블 조회 | ❌ 신 allcone 직접 호출 |
| **침범 없음** | Mapper가 `allcone_*` 또는 `chainportaldb.*` 테이블만 사용 | ❌ allcone 자체 데이터 |

### 테이블 소유 판정 규칙

| prefix | 소유 |
|--------|------|
| `allcone_*` | allcone 소유 |
| `general_*`, `terminal_*`, `container_*`, `tb_b_*`, `bctrans_*`, `tss_*`, `user_regist_truck` | bctrans 소유 |
| `chainportaldb.*` | 외부 DB (allcone 직접 접근 허용) |

### 복합 조건 주의

일부 엔드포인트는 **여러 침범 조건을 동시에 충족**한다. 대표적으로 침범 B(bctrans 테이블 DB 직통) + **bctrans 패키지 Service/Repository를 Bean 주입받아 내부 함수로 호출**하는 경우. 이 경우 bridge API 추가 대상 여부는 침범 B와 동일하게 판단하되, 복합임을 표 내 주석으로 명시한다.

## 6. 리스크/미확인

1. **V1 컨테이너 핸들 (`/mobile/terminal/req/container/handle/*`)**: 앱 RemoteConfig엔 V2만 있음. V1은 bctrans에 열려있으나 앱 미호출. V2가 HTTP로 V1 호출하는 구조.
2. **`user_regist_truck` 테이블 소유**: 접두사 규칙상 bctrans지만 실제 allcone 전용. 운영 정책 확정 필요.
3. **앱 외 호출 경로**: 배치/스케줄러/서버간 호출은 본 조사에 포함 안 됨.
4. **notice 단순 라벨 조인**: `general_terminal` 조인이 있으나 단순 터미널명 라벨. 신 allcone이 자체 캐시/복제 가능.
