# 민원 분석 — 취소된 인수도증이 계속 표출 (KOCU2219334 / 부산98사3975)

## 결론
게이트인 후 **게이트아웃 없이 취소만** 된 건이었고, 그 취소가 **기사 앱 화면에 갱신되지 않아** 이전 위치(3E-22)가 유효한 것처럼 남아 있었습니다.

| 15:53 | GateIn | 6J-15 |

## 원인
- 인수도증 조회는 **게이트아웃 유무로만 종료를 판정** → 게이트아웃 없는 취소건은 약 6시간 동안 활성으로 표출됨.
- 앱은 취소를 받으면 "취소 컨테이너"로 표시하고 위치를 숨김 → **갱신만 됐다면 3E-22는 사라졌어야 함.**
- 즉 **취소 시점에 앱이 재조회되지 않은 것**이 핵심.
- 앱은 **인수도증 화면 최초 진입 + 당겨서 새로고침** 시에만 재조회하고, 단순 앱 재개로 이전 화면이 복귀하면 재조회하지 않음.

## OS별
- **iOS**: 백그라운드에서 취소 알림을 무시한 뒤 앱을 다시 켜 이전 인수도증 화면이 그대로 복귀하면 → 갱신 안 됨(3E-22 잔존). **당겨서 새로고침 필요.**
- **Android**: 백그라운드 수신 시 갱신이 동작하도록 설계돼 있으나, 실제 동작은 **단말 재현 테스트로 확인 필요.**

## 13시 진입 정황
기사 진술은 "13시경 진입"이나 실제 게이트인은 11:14, 취소는 12:23. → 기사는 **이미 취소된 3E-22를 보고** 진입·대기한 것으로, 앱 미갱신 가설과 일치.

## 추가 확인 필요

- 단말 OS가 안드로이드일 경우 백그라운드 재조회 재현 테스트
- 11:14 게이트인 → 12:23 취소의 터미널 측 처리 경위

## 개선 방향 (참고)

- 취소를 **운송오더 삭제처럼 명시적 알림/표시**로 격상
