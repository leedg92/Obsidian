- vbs? tss? edi? 전자인수도증? copino
	- edi : 데이터를 주고받는 망
		- copino : 운송사가 요청한 운송건 (to 운송기사) == 운송오더

	- eslip : 터미널에서 발급하는 전자 인수도증
		- eslip에 대한 무브먼트정보는 edi을 타고 중계망 <-> TOS가 데이터를 주고받는다
  
	- vbs(vehicle booking service) : 운송예약서비스(기사 to 터미널), 언제갈거다 등등..(edi망을 타고 와서 적재된 운송건에 대한 예약)

	- tss(== itt) : 반출 + 반입(이때는 edi망을 타지않고, BaaS탄다), edi망을 통한 copino 두개를 묶어놓은 논리적 개념

- tss? group tss? (중계망이 블록체인, 단일 진입점)
	- tss : 운송사 -> bctrans -> BaaS / 운송사 -> BaaS (확인 필요)
	- group-tss: 운송사 -> bctrans -> BaaS

- 인프라간 프로세스 흐름
	1. 운송사가 edi망을 통해 copino를 tos로 전송한다
	2. TA가 tos에 도착한 copino 정보를 읽는다(읽는 방식은 다양)
	3. 읽은 정보를 정제를 거쳐 BaaS로 전송한다
	4. BaaS에서 고유키 생성 후에 TA와 bctrans로 invoke alarm 요청한다
	5. TA는 받은걸로 TOS에 기록(TA용 인터페이스 테이블 업데이트)
	6. bctrans는 받은걸로 내부 디비에 기록 후 알림
