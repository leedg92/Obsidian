# NICE 본인인증 앱 연동 가이드

## 개요

기존 SMS 인증(`/mobile/auth/req`, `/mobile/auth/res`)을 NICE 본인인증으로 대체합니다.
모든 본인인증이 필요한 기능(회원가입, 아이디찾기, 비밀번호재설정)이 동일한 인증 흐름을 사용합니다.

## 공통 인증 흐름

```
┌─────────┐       ┌──────────┐       ┌──────────┐       ┌──────────┐
│   앱    │       │  백엔드   │       │  NICE    │       │  백엔드   │
└────┬────┘       └────┬─────┘       └────┬─────┘       └────┬─────┘
     │                 │                  │                   │
     │ 1. auth-url 요청│                  │                   │
     │────────────────>│                  │                   │
     │                 │ 2. NICE API 호출 │                   │
     │                 │─────────────────>│                   │
     │   3. auth_url   │                  │                   │
     │<────────────────│                  │                   │
     │                 │                  │                   │
     │ 4. WebView 오픈 │                  │                   │
     │─────────────────────────────────-->│                   │
     │                 │                  │                   │
     │         5. 사용자 본인인증 수행     │                   │
     │                 │                  │                   │
     │                 │   6. return_url 콜백                 │
     │                 │<─────────────────│                   │
     │                 │ 7. 복호화 + certId 생성              │
     │                 │                  │                   │
     │   8. certId     │                  │                   │
     │<────────────────│                  │                   │
     │                 │                  │                   │
     │ 9. certId로 비즈니스 API 호출      │                   │
     │───────────────────────────────────────────────────────>│
     │  10. 처리 결과  │                  │                   │
     │<───────────────────────────────────────────────────────│
```

## 1단계: NICE 인증 URL 요청 (공통)

### Request
```
GET /mobile/auth/nice/auth-url?purpose={purpose}
```

| 파라미터 | 필수 | 설명 |
|---------|------|------|
| purpose | O | 인증 목적: `SIGN_UP`, `SEARCH_ID`, `CHANGE_PW`, `AUTH_UPDATE` |

### Response (성공)
```json
{
  "result": "success",
  "data": {
    "auth_url": "https://nice.checkplus.co.kr/..."
  }
}
```

### 앱 처리
- `auth_url`을 WebView로 오픈
- 사용자가 NICE 인증창에서 본인인증 수행

---

## 2단계: 인증 결과 수신 (공통)

NICE 인증 완료 후 `return_url`이 호출되며, 백엔드가 자동으로 처리합니다.

### Response (WebView에서 수신)
```json
{
  "result": "success",
  "data": {
    "certId": "a1b2c3d4e5f6..."
  }
}
```

### 앱 처리
- WebView에서 응답의 `certId`를 추출 (JS Bridge 또는 URL scheme)
- WebView 닫기
- `certId`를 이용하여 3단계 비즈니스 API 호출

> **주의**: certId의 유효 시간은 **10분**입니다. 만료 시 다시 인증해야 합니다.

---

## 3단계: 비즈니스 API 호출 (목적별)

### A. 회원가입 (purpose=SIGN_UP)

기존과 동일한 `/mobile/signup` API를 호출하되, `certId` 필드를 추가합니다.

```
POST /mobile/signup
Header: deviceToken: {deviceToken}
```

```json
{
  "certId": "a1b2c3d4e5f6...",
  "userId": "newuser01",
  "password": "password123!",
  "serviceTermsAgreement": "Y",
  "personalInfoProcessingPolicyAgreement": "Y",
  "personalLocationInfoProcessingPolicyAgreement": "Y",
  "locationInfoBusinessTermsAgreement": "Y",
  "agreeMarketing": "N",
  "recommendTruckNo": "",
  "recommendName": ""
}
```

> **변경사항**: 기존에 직접 전달하던 `userName`, `userHp`는 `certId`로 대체됩니다.
> 백엔드가 certId로 NICE 인증 결과에서 이름, 전화번호, 생년월일, CI를 자동으로 가져옵니다.

---

### B. 아이디 찾기 (purpose=SEARCH_ID)

```
POST /mobile/user/findId
```

#### Request Body
```json
{
  "certId": "a1b2c3d4e5f6..."
}
```

#### Response (성공)
```json
{
  "result": "success",
  "data": {
    "userId": "new****01",
    "userName": "홍길동",
    "joinDate": "2026-03-26 12:15"
  }
}
```

#### Response (실패 - 미등록 회원)
```json
{
  "result": "error",
  "message": "등록된 회원 정보를 찾을 수 없습니다."
}
```

> **참고**: userId는 마스킹 처리되어 반환됩니다 (앞3자 + **** + 뒤2자).

---

### C. 비밀번호 재설정 (purpose=CHANGE_PW)

#### Step 1: NICE 인증으로 certId 획득 (위 1~2단계)

#### Step 2: 비밀번호 재설정 API 호출

```
POST /mobile/user/resetPassword
```

##### Request Body
```json
{
  "certId": "a1b2c3d4e5f6...",
  "userId": "newuser01",
  "password": "newPassword123!"
}
```

##### Response (성공)
```json
{
  "result": "success"
}
```

##### Response (실패 - 아이디 불일치)
```json
{
  "result": "error",
  "message": "아이디가 본인 인증 정보와 일치하지 않습니다."
}
```

---

### D. 기존 회원 인증 업데이트 (purpose=AUTH_UPDATE)

기존 API 그대로 사용합니다.

```
POST /mobile/user/niceAuthUpdate
Header: deviceToken: {deviceToken}
```

```json
{
  "certId": "a1b2c3d4e5f6...",
  "userId": "existingUser01"
}
```

---

## 기존 API 대비 변경 요약

| 기능               | 기존 (SMS 인증)                                              | 변경 후 (NICE 인증)                                                 |
| ---------------- | -------------------------------------------------------- | -------------------------------------------------------------- |
| 인증 요청            | `POST /mobile/auth/req`                                  | `GET /mobile/auth/nice/auth-url?purpose=...`                   |
| 인증 확인            | `POST /mobile/auth/res`                                  | WebView → certId 수신 → 비즈니스 API 호출                              |
| 아이디 찾기           | `/mobile/auth/res` (authType=FIND_ID)                    | `POST /mobile/user/findId` (certId)                            |
| 비밀번호 재설정         | `/mobile/auth/res` → `/mobile/user/modpwd`               | `POST /mobile/user/resetPassword` (certId + userId + password) |
| 회원가입 인증          | `/mobile/auth/res` (authType=SIGN_UP) → `/mobile/signup` | NICE 인증 → `/mobile/signup` (certId 포함)                         |
| 비밀번호 변경 (로그인 상태) | `POST /mobile/user/modpwd`                               | **변경 없음** (기존 그대로)                                             |
- 참고 : 인증 요청 purpose값 = `ALLCONE_APP_RESET_PASSWORD / ALLCONE_APP_FIND_ID / ALLCONE_APP_JOIN`
## 에러 코드

| 에러 메시지 | 원인 | 해결 |
|------------|------|------|
| 인증 URL 요청에 실패했습니다. | NICE 서버 통신 오류 | 재시도 |
| 인증 결과 조회에 실패했습니다. | 복호화 실패 또는 위변조 | 재인증 |
| 유효하지 않은 인증 정보입니다. | certId 만료(10분) 또는 이미 사용됨 | 재인증 |
| 등록된 회원 정보를 찾을 수 없습니다. | CI_HASH 매칭 실패 (미가입 또는 NICE 미인증 기존 회원) | 회원가입 안내 |
| 아이디가 본인 인증 정보와 일치하지 않습니다. | 입력한 userId와 CI 매칭 불일치 | userId 재확인 |

## WebView에서 certId 추출 방법 (참고)

return_url 콜백 응답이 JSON 형태로 반환되므로, WebView에서 다음 방법으로 certId를 추출할 수 있습니다:

1. **JS Bridge**: WebView의 JavaScript에서 네이티브 함수 호출
2. **URL 감지**: WebView의 URL 변경을 감지하여 return_url 패턴 매칭
3. **Response Body 파싱**: WebView에서 응답 본문의 JSON을 파싱
