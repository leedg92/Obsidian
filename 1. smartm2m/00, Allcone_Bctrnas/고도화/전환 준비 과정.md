## Phase 1: 데이터 흐름 파악

> 목표: 현재 시스템의 모든 외부 연동 지점과 데이터셋을 파악한다.

| 단계 | 내용 | 상태 |
|------|------|------|
| 1-1 | 외부 진입점 전수 조사 (어떤 외부 시스템이 뭘 호출하는지) | ✅ 완료 |
| 1-2 | 각 진입점별 요청/응답 데이터셋 정리 | ✅ 완료 |
| 1-3 | tss/vbs 분기 조건 정리 (어떤 기준으로 나뉘는지) | ✅ 완료 |
| 1-4 | 내부→외부 아웃바운드 호출 지점 전수 조사 | ✅ 완료 |
| 1-5 | 각 아웃바운드 호출별 데이터셋 + 호출 주체(vbs/tss/공통) 정리 | ✅ 완료 |
| 1-6 | 앱 API 전수 조사 (allcone 엔드포인트 목록) | 미착수 |
| 1-7 | 앱 API별 뒷단 호출 흐름 (자체 처리 vs 포워딩) | 미착수 |
| 1-8 | FCM 발송 트리거 정리 (어떤 이벤트가 앱 푸시를 일으키는지) | 미착수 |

> 결과물: [Phase 1 분석 결과](msa-phase1-data-flow.md)
