## 3. TSS/VBS 분기 조건 정리

### 3-1. 인바운드 분기

| 분기 기준 | 설명 |
|----------|------|
| **URL 경로** | `/itt/invoke/alarm` = TSS, `/vbs/invoke/alarm` = VBS |
| **URL 경로** | `/tss/*` = TSS 전용, `/vbs/*` = VBS 전용 |
| **serverName** | invoke 컨트롤러 내부에서 allcone/bctrans 처리 분기 |

### 3-2. 아웃바운드 분기

| 외부 시스템 | TSS 전용 | VBS 전용 | 공통 |
|------------|---------|---------|------|
| 블록체인 | CreateCopino, CreateCopinoWithGroup, GetCopino, GetGroupCopino, GetTransHistory, UpdateConInfo | GetVbsTransHistory | BChainService.exchange |
| 터미널 에이전트 | fetchMissingMovements | - | - |
| 터미널 직접 | 검증(REQ_VERIFY_ITT), 그룹시작/취소, 그룹수정 | 방향전환(034), 교체(035) | 게이트인/아웃 |
| AAS 토큰서버 | 토큰검증, 콜백URL조회 | - | - |
| 트럭커 콜백 | 터미널이벤트/상태변경/검증결과 알림 | - | - |
| 체인포탈 | - | - | 전부 공통 |

### 3-3. 핵심 발견

1. **게이트인/아웃은 VBS/TSS 동일한 터미널 API 사용** → transport-handler 또는 공통 처리 가능
2. **블록체인 BChainService.exchange()는 VBS/TSS 양쪽에서 사용** → 공통 모듈 후보
3. **TSS가 외부 연동이 훨씬 많음** (AAS, 트럭커 콜백, 터미널에이전트 보정 등)
4. **VBS 외부 연동은 블록체인 + 터미널 직접이 전부** → 상대적으로 단순
