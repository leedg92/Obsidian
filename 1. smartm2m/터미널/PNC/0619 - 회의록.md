## 1. 게이트 보정 관련 처리 코드

### 처리 코드 기준

|코드|의미|사용 시점|
|---|---|---|
|C|취소|반입 컨테이너 취소|
|A|반입|반입 정상 진입|
|I|반입 (초기·복화)|최초 진입 또는 복화 처리|
|R|반출|반출 무조건 R (보정 포함)|

### 케이스별 처리

**반출**

- 정상 → R
- 진입했으나 방문번호 없고 조회도 안 됨 → I
- 진입했으나 에러 → R
- 20자 컨테이너 2개 중 하나 인식 불가 → R

**반입**

- 정상 → A (또는 I)
- 취소 → C
- 콤바인(앞뒤 컨테이너) → conNo1 / conNo2 구분 처리
- 콤바인 중 하나 인식 불가 → 해당 건 C 처리 후 반출(R)로 구분

**방문번호(비짓번호)**

- 진입 이력 있으면 포함 전송
- 이력 없으면 빈값 → 추가 처리 필요

**기사 사전정보 선택 (앞/뒤)**

- 게이트 통과 후 기사가 직접 선택 (기사 본인만 아는 정보)

## 3. 올컨e 사전정보 처리 로직

**키값 기준**

- 기본: 차량번호 + 컨테이너번호로 최신 1건 유지
- 동일 키로 다른 정보 수신 시 최신으로 덮어쓰기
- 수신 경로 무관, TR 폴더에 정해진 형식으로 투입하면 처리됨
- 터미널마다 키 방식 상이할 수 있음 → 확인 필요

**플래그 (미지정 공컨 한정)**

- P: 사용 중 / D: 완료 / C: 취소

**공컨 반출**

- `EMPTY1` + `plan_cont`(예상 컨테이너 번호) 조합으로 처리
- 교체 발생 시 `plan_cont`만 변경

**Seq(시퀀스)**

- 불필요 (PNC 강민모 차장 의견) — 진입·퇴장 시 새로운 사이클이므로 이전 seq 참조 불필요

**취소 처리**

- 취소 시 사전정보 함께 삭제 여부 선택 가능
    - 삭제 선택 → 사전정보 테이블 삭제 → 리무브 코피노 이벤트 → 후속 처리
    - 삭제 미선택 → 사전정보 유지
- 게이트아웃 완료 = 사전정보 사용 완료 → 재진입 시 재생성 필요
- 실제 미션 수행 전 취소된 경우 → 미션 그대로 유지 가능

**그룹00가 (임시 차량번호)**

- 할당 완료 시 실제 차량번호 + 실제 컨테이너번호로 교체
- 취소 시 그룹00가 롤백 또는 삭제 (정책 확인 필요)
